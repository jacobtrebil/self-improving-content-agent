## Strategy memory — apply while generating
_Distilled from 6 past run(s). Higher support = more runs backed this lesson._

- **[structure] Validate CTA, image prompts, and JSON schema before accepting any generated deck** — All 30 decks in this batch failed has_cta, follows_json_schema, and has_image_prompts — a 100% failure rate indicating the generator is not reliably producing schema-compliant output. Before the next batch: (1) confirm the prompt explicitly requires a CTA slide pointing to the app screenshots, (2) confirm cover_query is populated on every deck, (3) run the code-eval checks on the first 2-3 generated decks as a smoke test before generating all 30, and (4) treat any missing field as a generation failure requiring a retry rather than a manual patch.  _(support 1, confidence 0.98)_
- **[hook] Open with a hook that challenges an assumption, not one that states a benefit** — Hooks like 'A short nap can reset your afternoon' or 'Eat the rainbow, age more slowly' are soft promises — they confirm what viewers hope is true. Rewrite every hook to create mild cognitive dissonance or a knowledge gap: something the viewer thinks they understand but now doubts. Test each hook against: 'would someone who already knows this topic stop scrolling?' If yes, keep it. If no, add a specific number, a counterintuitive qualifier, or a reframe that makes the claim feel slightly dangerous or unexpected.  _(support 1, confidence 0.9)_
- **[novelty] Explicitly blacklist the most-recycled topic frames when prompting** — The following angles are so saturated they scored ≤2 on novelty and should be blocked or radically reframed in any future batch: 'eat the rainbow,' 'take the stairs,' 'magnesium for sleep,' 'screen time before bed,' 'core protects your back,' 'gratitude journaling,' 'cold exposure = resilience,' and 'eat more protein.' If a topic from this list must appear, require a specific and surprising sub-angle (a mechanism, a counterintuitive population finding, an underreported nuance) that the generic version of the advice omits. The topic name alone is not enough justification to include it.  _(support 1, confidence 0.88)_
